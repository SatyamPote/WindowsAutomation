## What's Changed

- **Truly standalone install** — `Lotus.app` now bundles a universal `uv` binary and the full bot runtime template. Drag the app to `/Applications`, launch, and the first-run wizard provisions Python + dependencies into `~/Library/Application Support/Lotus/runtime/` automatically. **No `git clone`, no `uv sync`, no Homebrew required.**
- **Universal binary** — single `arm64 + x86_64` build, runs natively on both Apple Silicon and Intel Macs.
- **New DMG installer** — opens to a styled drag-to-install window with a lotus-pond banner background and an `Applications` shortcut.
- **Menu-bar Toggle Bot** works again — fixed a stale rename that crashed the menu when starting/stopping the bot service.

## ⚠️ Breaking changes

- The bot runtime moved from the `Mac-MCP/` source checkout to `~/Library/Application Support/Lotus/runtime/`. Anyone running v1.x with a manually-installed `.venv` next to `bot_service.py` will get a fresh runtime on first launch of v2.0.0 — the LaunchAgent's `ProgramArguments` are rewritten to point at the new location. The old source checkout is no longer required and can be deleted.
- Bundle layout changed: `Resources/bin/uv` and `Resources/runtime-template/` are new top-level entries inside `Lotus.app`. Any external tool that scraped paths from the bundle will need to be updated.

## ✨ New

- **Bundled uv** — `Lotus.app/Contents/Resources/bin/uv` ships a universal `uv 0.5.4` binary used by the installer wizard and the launchd service.
- **Runtime template** — `bot_service.py`, `pyproject.toml`, `uv.lock`, and the `mac_mcp` Python package are staged at `Lotus.app/Contents/Resources/runtime-template/` and rsynced into a writable runtime dir on first launch.
- **Writable runtime dir** at `~/Library/Application Support/Lotus/runtime/` — keeps the signed app bundle immutable while still letting `uv sync` create a venv.
- Universal `Lotus.app` binary (arm64 + x86_64) verified at build time via `lipo -archs`.
- DMG layout: 640×639 window with banner artwork, icons centered over the pond.
- Per-release `release-notes/v<version>.md` files become the curated "What's Changed" section on GitHub Releases.

## 🐛 Fixes

- **App couldn't run from `/Applications`** because `InstallManager` tried to create `.venv` inside the read-only signed app bundle. Now writes to `~/Library/Application Support/Lotus/runtime/` instead.
- **Menu bar → Toggle Bot crashed** because `AppDelegate` called `state.startAgent()` / `state.stopAgent()`, methods that no longer exist on `AppState`. Now correctly invokes `startBot()` / `stopBot()`.

## 🔧 Internals

- **GitHub Actions release pipeline** (`.github/workflows/swift.yml`) — builds the universal app, packages the DMG, attaches both to a GitHub Release with auto-generated PR notes underneath the curated section.
- **Two release paths**:
  - Push a `v*` tag → automatic release.
  - Or click **Run workflow** in the Actions tab with `publish=true` and a version → tags + releases without leaving the browser.
- **Manual-only triggers** — the workflow no longer runs on every push to `main` or PR. Releases are intentional, not incidental.
- Build runs on `macos-15` with Xcode 16 / Swift 6 pinned via `setup-xcode`.
- DMG build script auto-skips Finder window styling on headless CI runners (the DMG is still produced, just without the custom icon positions).
- Verification steps assert the universal binary slices are present and the DMG mounts cleanly with `Lotus.app` + `Applications` symlink inside.

## Install

1. Download `Lotus-2.0.0.dmg` below and drag **Lotus.app** to **Applications**.
2. Launch Lotus from `/Applications` (the 🌸 icon appears in your menu bar).
3. The first-run wizard provisions Python 3.13 + bot dependencies into `~/Library/Application Support/Lotus/runtime/` (~30–60 seconds, one-time).
4. Enter your Telegram token + Ollama model and hit **Save & Launch Bot**.

That's it — no source checkout, no `uv sync`, no system Python required.

> The DMG is ad-hoc signed (not notarized). On first launch, right-click → **Open**, or run:
> ```bash
> xattr -d com.apple.quarantine /Applications/Lotus.app
> ```

### Upgrading from v1.0.0

Replace `Lotus.app` in `/Applications`. Your config (`~/Library/Application Support/Lotus/config.json`) and the `com.lotus.botservice` LaunchAgent are untouched. On next launch, Lotus will populate the new `runtime/` subdirectory and install the venv there. The old `Mac-MCP/` source checkout (if any) is no longer used by the app — you can delete it.

```bash
# After replacing Lotus.app, restart the service from the menu bar,
# or kick it from the CLI:
launchctl kickstart -k gui/$(id -u)/com.lotus.botservice
```
