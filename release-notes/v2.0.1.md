## What's Changed

Patch release — fixes a crash on launch introduced in v2.0.0 when opening the control panel. v2.0.0 users should upgrade.

## 🐛 Fixes

- **Crash on launch / control panel not loading** — `make_app.sh` was only copying the `Lotus` binary into the bundle and skipping the SPM-generated `Lotus_Lotus.bundle`. Any `Bundle.module` access then fired a fatal `_assertionFailure` (`EXC_BAD_INSTRUCTION`), which surfaced the moment `ControlPanelView` tried to resolve the logo path. The build script now copies `Lotus_Lotus.bundle` next to the binary in `Contents/MacOS/` and the app boots cleanly.
- **Missing logos / placeholder 🌸 in the header** — `assets/` was no longer reachable from the installed app. `make_app.sh` now stages it at `Contents/Resources/assets/`, and `AppConfig.assetsDir` reads from `Bundle.main` instead of `Bundle.module` so the lookup survives app translocation.

## Install

1. Download `Lotus-2.0.1.dmg` below and drag **Lotus.app** to **Applications**.
2. Launch Lotus from `/Applications` (the 🌸 icon appears in your menu bar).
3. If you already ran v2.0.0 once, the runtime under `~/Library/Application Support/Lotus/runtime/` is reused — no re-install needed.

> The DMG is ad-hoc signed (not notarized). On first launch, right-click → **Open**, or run:
> ```bash
> xattr -d com.apple.quarantine /Applications/Lotus.app
> ```

### Upgrading from v2.0.0

Replace `Lotus.app` in `/Applications`. Your config and the existing runtime venv are untouched. Restart the bot from the menu bar (or `launchctl kickstart -k gui/$(id -u)/com.lotus.botservice`) to pick up any path changes.
